◆ Thread_phase

enum Thread_phase

Thread phase.

The phase of a thread denotes the state of its context stored in the TCB of the thread. A CTX_CLEAN thread means that, the context stored in the TCB is up-to-date. In this case, it is legal to swap context to this thread. A CTX_DIRTY thread marks the case when the thread was, or still is, being executed at some core, therefore its stored context should not be used.

The following invariant of the scheduler guarantees correctness:

A TCB is in the scheduler queue, if and only if, its Thread_state is READY and the Thread_phase is CTX_CLEAN.

◆ sleep_releasing()

void sleep_releasing ( Thread_state  newstate,
Mutex *  mx,
enum SCHED_CAUSE  cause,
TimerDuration  timeout 
)

Block the current thread.

This call will block the current thread, changing its state to STOPPED or EXITED. Also, the mutex mx, if not NULL, will be unlocked, atomically with the blocking of the thread.

In particular, what is meant by 'atomically' is that the thread state will change to newstate atomically with the mutex unlocking. Note that, the state of the current thread is RUNNING. Therefore, no other state change (such as a wakeup, a yield, another sleep etc) can happen "between" the thread's state change and the unlocking.

If the newstate is EXITED, the thread will block and also will eventually be cleaned-up by the scheduler. Its TCB should not be accessed in any way after this call.

A cause for the sleep must be provided; this parameter indicates to the scheduler the source of the sleeping operation, and can be used in scheduler heuristics to adjust scheduling decisions.

A timeout can also be provided. If the timeout is not NO_TIMEOUT, then the thread will be made ready by the scheduler after the timeout duration has passed, even without a call to wakeup() by another thread.

◆ spawn_thread()

TCB* spawn_thread ( PCB *  pcb,
void(*)()  func 
)

Create a new thread.

This call creates a new thread, initializing and returning its TCB. The thread will belong to process pcb and execute func. Note that, the new thread is returned in the INIT state. The caller must use wakeup() to start it.

Parameters
pcbThe process control block of the owning process. The scheduler simply stores this value in the new TCB, and otherwise ignores it
funcThe function to execute in the new thread.
Returns
A pointer to the TCB of the new thread, in the INIT state.
