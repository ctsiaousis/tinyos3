1 # Building tinyos using make
2 
3 To build your code easily, a complex Makefile is provided. This manual presents several ways to run
4 make using this Makefile.
5 
6 ## Basic building
7 
8 Just type the command
9 ```
10 $ make
11 ```
12 This command will compile all the files and build all the executable programs and the examples.
13 The built programs will contain debugging information, so that you can run the debugger on them.
14 
15 ## Recompiling all the files
16 
17 When there are weird errors, it is possible that something was corrupted. To remove all binary files
18 as well as all named pipes, give the following command:
19 ```
20 $ make clean
21 ```
22 The you can rebuild everything by giving the command
23 ```
24 $ make
25 ```
26 
27 ## Building with full optimizations on
28 
29 To see how fast your code is, you can build with full optimizations. Give the following:
30 ```
31 $ make DEBUG=0 clean all
32 ```
33 
34 ## Using valgrind
35 
36 If you have not installed valgrind, the code will be built without support for it. But valgrind is very
37 useful for debugging. You can install it with the following command:
38 ```
39 $ sudo apt install valgrind
40 ```
41 
42 ## Re-making the dependencies
43 
44 When you change the \#include headers in some file, you should rebuild the dependencies.
45 ```
46 $ make depend
47 ```
48 This way you are sure that make re-builds everything it needs to rebuild every time.

51 ## Building the documentation
52 
53 The code of tinyos comes with a lot of useful documentation. To build it, you need a program called
54 'doxygen'. Then, you can view the documentation in your browser. Here is how to install doxygen
55 ```
56 $ sudo apt install doxygen graphvis
57 ```
58 Then, you can build the documentation, by the following command:
59 ```
60 $ make doc
61 ```
